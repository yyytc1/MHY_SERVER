API调用说明

概述

内部账号管理系统提供RESTful API接口，用于管理账号信息。所有API都基于HTTP协议，支持JSON格式的数据交换。

基础信息
  • 基础URL: http://localhost:8080/api
  • 数据格式: JSON
  • 字符编码: UTF-8
  • API版本: v1

认证

当前版本无需认证，所有API接口均可直接访问。未来版本可能会添加JWT或API Key认证机制。

API端点

POST

/api/accounts

创建新账号

请求参数:
{
  "username": "string (必填, 最大50字符, 唯一)",
  "password": "string (必填, 最大255字符)",
  "account_status": "string (可选, 默认: 'correct')",
  "invite_status": "string (可选, 默认: 'pending')",
  "account_token": "string (可选, 无限制长度)"
}
响应示例:
HTTP 201 Created
{
  "message": "账号创建成功",
  "data": {
    "id": 1,
    "username": "example_user",
    "password": "password123",
    "account_status": "correct",
    "account_token": "",
    "invite_status": "pending",
    "created_at": "2025-01-27T10:00:00Z",
    "updated_at": "2025-01-27T10:00:00Z"
  }
}
GET

/api/accounts

获取账号列表（支持搜索和分页）

查询参数:
  • search - 搜索关键词（用户名或密码）
  • page - 页码（默认: 1）
  • page_size - 每页数量（默认: 10，最大: 100）
响应示例:
HTTP 200 OK
{
  "data": [
    {
      "id": 1,
      "username": "example_user",
      "password": "password123",
      "account_status": "correct",
      "account_token": "",
      "invite_status": "pending",
      "created_at": "2025-01-27T10:00:00Z",
      "updated_at": "2025-01-27T10:00:00Z"
    }
  ],
  "total": 1,
  "page": 1,
  "page_size": 10,
  "total_pages": 1
}
GET

/api/accounts/{id}

根据ID获取单个账号信息

路径参数:
  • id - 账号ID（必填）
响应示例:
HTTP 200 OK
{
  "data": {
    "id": 1,
    "username": "example_user",
    "password": "password123",
    "account_status": "correct",
    "account_token": "",
    "invite_status": "pending",
    "created_at": "2025-01-27T10:00:00Z",
    "updated_at": "2025-01-27T10:00:00Z"
  }
}
GET

/api/accounts/pending

获取状态为pending的单个账号资料

响应示例:
HTTP 200 OK
{
  "data": {
    "id": 1,
    "username": "example_user",
    "password": "password123",
    "account_status": "correct",
    "account_token": "",
    "invite_status": "pending",
    "created_at": "2025-01-27T10:00:00Z",
    "updated_at": "2025-01-27T10:00:00Z"
  }

HTTP 404 Not Found
{
  "error": "没有找到状态为pending的账号"
}
PUT

/api/accounts/{id}

更新指定ID的账号信息

路径参数:
  • id - 账号ID（必填）
请求参数:
{
  "username": "string (可选, 最大50字符, 唯一)",
  "password": "string (可选, 最大255字符)",
  "account_status": "string (可选)",
  "invite_status": "string (可选)",
  "account_token": "string (可选, 无限制长度)"
}
响应示例:
HTTP 200 OK
{
  "message": "账号更新成功",
  "data": {
    "id": 1,
    "username": "updated_user",
    "password": "newpassword123",
    "account_status": "correct",
    "account_token": "token123",
    "invite_status": "invited",
    "created_at": "2025-01-27T10:00:00Z",
    "updated_at": "2025-01-27T11:00:00Z"
  }
}
DELETE

/api/accounts/{id}

删除指定ID的账号

路径参数:
  • id - 账号ID（必填）
响应示例:
HTTP 200 OK
{
  "message": "账号删除成功"
}
GET

/api/accounts/export

导出所有账号数据为CSV格式

响应示例:
HTTP 200 OK

返回CSV文件下载，包含以下列：

  • ID
  • 用户名
  • 密码
  • 账号状态
  • 邀约状态
  • 创建时间
  • 更新时间

数据模型

Account（账号模型）

字段名 类型 约束 描述
id uint 主键，自增 账号唯一标识
username string 最大50字符，唯一索引，非空 用户名
password string 最大255字符，非空 密码
account_status string 最大20字符，默认'correct' 账号状态：correct(密码正确), incorrect(密码错误), risk(登录风险)
account_token string 无限制（TEXT类型） 账号令牌
invite_status string 最大20字符，默认'pending' 邀约状态：pending(待处理), invited(已邀约), not_invited(未邀约)
created_at time.Time 自动生成 创建时间
updated_at time.Time 自动更新 更新时间

示例代码

JavaScript (Fetch API)

// 获取所有账号
fetch('http://localhost:8080/api/accounts')
  .then(response => response.json())
  .then(data => console.log(data));

// 创建账号
fetch('http://localhost:8080/api/accounts', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    username: 'new_user',
    password: 'password123',
    account_status: 'correct',
    invite_status: 'pending'
  })
})
.then(response => response.json())
.then(data => console.log(data));

// 获取pending状态账号
fetch('http://localhost:8080/api/accounts/pending')
  .then(response => response.json())
  .then(data => console.log(data));

// 更新账号
fetch('http://localhost:8080/api/accounts/1', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    account_status: 'invited',
    invite_status: 'invited'
  })
})
.then(response => response.json())
.then(data => console.log(data));

// 删除账号
fetch('http://localhost:8080/api/accounts/1', {
  method: 'DELETE'
})
.then(response => response.json())
.then(data => console.log(data));

cURL

# 获取所有账号
curl -X GET "http://localhost:8080/api/accounts"

# 创建账号
curl -X POST "http://localhost:8080/api/accounts" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "new_user",
    "password": "password123",
    "account_status": "correct",
    "invite_status": "pending"
  }'

# 获取pending状态账号
curl -X GET "http://localhost:8080/api/accounts/pending"

# 更新账号
curl -X PUT "http://localhost:8080/api/accounts/1" \
  -H "Content-Type: application/json" \
  -d '{
    "account_status": "invited",
    "invite_status": "invited"
  }'

# 删除账号
curl -X DELETE "http://localhost:8080/api/accounts/1"

Python (requests)

import requests
import json

base_url = "http://localhost:8080/api"

# 获取所有账号
response = requests.get(f"{base_url}/accounts")
print(response.json())

# 创建账号
data = {
    "username": "new_user",
    "password": "password123",
    "account_status": "correct",
    "invite_status": "pending"
}
response = requests.post(f"{base_url}/accounts", json=data)
print(response.json())

# 获取pending状态账号
response = requests.get(f"{base_url}/accounts/pending")
print(response.json())

# 更新账号
update_data = {
    "account_status": "invited",
    "invite_status": "invited"
}
response = requests.put(f"{base_url}/accounts/1", json=update_data)
print(response.json())

# 删除账号
response = requests.delete(f"{base_url}/accounts/1")
print(response.json())

错误码说明

状态码 说明 示例
200 请求成功 获取数据成功
201 创建成功 账号创建成功
400 请求参数错误 无效的ID格式
404 资源不存在 账号不存在
409 冲突 用户名已存在
500 服务器内部错误 数据库操作失败
